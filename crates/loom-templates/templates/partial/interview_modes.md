## Interview Modes

The user may request one of these structured sub-modes at any point in the interview. Phrase matching is loose — respond to intent, not exact wording.

- **"one by one"** (also: "let's go through one by one", "go through them one at a time", etc.) — When you have multiple open design questions, ask exactly one question per turn: a single design question. For each question, propose a suggested default with a short rationale, then wait for the user's prose answer before moving to the next. Optimizes for user attention: small decisions per turn, defaults ready to rubber-stamp or adjust.

- **"polish the spec"** (also: "polish this spec", "give it a polish", "do a polish", "do a polish pass", etc.) — Run a report-only spec/doc review: read the full spec end-to-end and report on readability issues (unclear phrasing, missing context), consistency issues (contradictions between sections, terminology drift), ambiguities (statements that could be read multiple ways), and structural problems (misplaced content, missing sections). Propose specific edits or findings, but do not apply edits unless explicitly asked to apply them. Typically run at end of a planning session, but available at any point.

Both modes remain within planning-only scope — no code changes. A polish pass is report-only unless the user explicitly asks for application.
